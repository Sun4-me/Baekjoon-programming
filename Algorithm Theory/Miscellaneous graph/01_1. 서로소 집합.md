# 서로소 집합
- 공통 원소가 없는 두 집합을 의미한다.
    - {1, 2}와 {3, 4}는 서로소 관계이다.

## 서로소 집합 자료구조
- 서로소 부분 집합들로 나누어진 원소들의 데이터를 처리하기 위한 자료구조
- 두 종류의 연산을 지원
  - 합집합(Union): 두 개의 원소가 포함된 집합을 하나로 합치는 연산
  - 찾기(Find): 특정한 원소가 속한 집합이 어떤 집합인지 알려주는 연산

  
## 동작 과정
1. 합집합 연산을 하나씩 확인하여, 서로 연결된 두 노드 A, B를 확인
    1. A와 B의 루트 노드 A', B'를 각각 찾는다.
    2. A'를 B'의 부모 노드로 설정한다.
2. 모든 합집합 연산을 처리할 때까지 1번의 과정을 반복한다.

## 연결성
- 연결성을 통해 손쉽게 집합의 형태를 확인 가능하다.
- 루트 노드에 즉시 접근 할수는 없다.
  - 루트 노드를 찾기 위해 부모 테이블을 계쏙해서 확인하며 거슬러 올라가야 한다.

## 문제점
- 유니온 연산이 편향될 경우 파인드 함수가 비효율적으로 동작한다.
- 최악의 경우 모든 노드를 다 확인하게 되어 시간 복잡도가 O(V)가 된다.

## 경로 압축
- 파인드 함수를 최적하기 위한 방법으로 경로 압축을 이용할 수 있다.
  - 파인드 함수를 재귀적으로 호출 한 뒤에 부모 테이블 값을 바로 갱신한다.
- 경로 압축기법을 정용하면 파인드 함수 호출 후 해당 노드의 루트 노드가 바로 부모 노드가 된다.

## 사이클 판별
- 무방향 그래프 내에서의 사이클을 판별할 때 사용 가능하다.
- 동작 과정
  1. 각 간선을 하나씩 확인하며 두 노드의 루트 노드를 확인한다.
     1.  루트 노드가 서로 다르다면 두 노드에 대해 합집합 연산을 수행한다.
     2.  루트노드가 서로 같다면 사이클이 발생한것이다.
  2. 그래프에 포함되어 있는 모든 간선에 대하여 1번 반복
